# pago-go — Agent Guide

`github.com/pago-sh/pago-go` (main type `Pago`) — the official pago.sh SDK for Go. **Generated by Speakeasy** from the pago.sh OpenAPI spec.

## ⚠️ Generated code
Most files are generated and **overwritten on regeneration** — do not hand-edit them. Source of truth: the OpenAPI spec + `.speakeasy/gen.yaml` (SDK `version`); the version constant lives in `pago.go` (`SDKVersion`). Regeneration is manual (CI workflows removed).

## Quick commands
```bash
go build ./...
go test ./...
go vet ./...
```
Go modules are versioned by **git tag** — release with `git tag v1.0.0 && git push origin v1.0.0` (there is no registry to publish to).

## Structure
- `*.go` (root) — generated SDK · `models/`, `internal/` — generated · `.speakeasy/` — gen.yaml + overlays

## Git Workflow

Branch model — the same across every repo in the pago.sh ecosystem:

- **`main`** — production. Releases (git tags) run from `main`. Never commit or push directly to `main`.
- **`develop`** — integration & testing. Validated here before reaching `main`.
- **`feat/<name>`** / **`fix/<name>`** — branch off `develop`, merge back into `develop`.

**Flow:** `feat/*` or `fix/*` → **`develop`** (test) → merge into **`main`** → production.
